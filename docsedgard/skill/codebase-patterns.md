# Codebase Patterns

## Reusable Patterns Extracted
- Server API routes for chat and attachment analysis.
- OpenAI direct fetch avoids adding SDK dependency where not necessary.
- Skill registry/router/system prompt separation is reusable.
- Memory loader should keep prompt compact and route by domain.
- File preview and upload should keep real File object plus metadata.
- Use honest fallback when provider/model/parser fails.

## Rewrite Risks
- Legacy branches may contain partially merged or superseded behavior.
- Some modules are demo-like and should be rebuilt as Copilot tools.
- Historical PR docs should not become current truth.

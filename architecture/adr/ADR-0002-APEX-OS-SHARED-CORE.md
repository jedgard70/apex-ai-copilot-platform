# Apex OS como Core compartilhado

**Status:** Aceito para Foundation; revisável
**Versão:** 0.1
**Data:** 2026-07-20

## Contexto

Produtos precisam reutilizar autoridades sem acoplamento.

## Decisão

Adotar Core independente e Shared Services contratuais.

## Alternativas

Monólito; duplicar Core por produto.

## Consequências positivas

Reuso e governança.

## Consequências negativas

Contratos exigem disciplina.

## Riscos

Core virar superdomínio.

## Critérios de revisão

Revisar quando evidência de mercado, segurança, custo, capacidade da equipe ou compatibilidade demonstrar que a decisão não cumpre seus objetivos.

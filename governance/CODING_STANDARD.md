# Padrão de código
**Status:** Foundation | **Versão:** 0.1 | **Data:** 2026-07-20

Código futuro deve ser legível, pequeno, tipado quando a linguagem permitir, sem segredos, com erros explícitos e interfaces estáveis. Nomes expressam domínio; comentários explicam decisões. Formatação/lint serão definidos por stack via ADR. Mudanças incluem teste proporcional, observabilidade e documentação. Código gerado recebe revisão e origem.
